# Command Center Prompt

Design the `Command Center` workspace for Prism as the runtime posture and urgent signal surface.

Goals:

- surface the most important operational issues first
- summarize posture without becoming a BI dashboard
- link directly into investigations, routes, providers, and changes

Include:

- posture summary
- active signals and investigations
- route and provider pressure
- config risk and change watch windows
- external evidence source health such as runtime, SLS, and OTLP

Constraints:

- one dominant region only
- avoid equal-weight cards everywhere
- make the urgent queue and drill-down path obvious
- preserve space for future ownership, acknowledgements, and runbooks
